Usage Guide

Creating new CVSS vectors
Using the buttons and input field at the top, you can:
  • Click on the first button that says CVSS to import multiple vectors at once.
  • Click on of the buttons next to it to add a vector of that CVSS version.
  • Input a CVSS vector of any version in the input field and press enter.
  • Input a CVE-ID and press enter to fetch the vectors of this CVE from the NVD.
Editing CVSS vectors
Click on a vector in the list to select it. A small arrow will appear to the left of the selected vector. You can directly edit the vector in the above text field, but below, you'll find a series of accordion elements, each representing a different segment of the CVSS vector, which allow for more convenient editing. Expand an accordion section by clicking its header to view and modify the individual components of that vector segment.
Interpreting the results
The table below the vector list shows the available calculated scores for each CVSS vector. Certain CVSS versions do not normalize all their scores to a 0-10 scale before displaying them. For example, the Exploitability Score of CVSS:3.1 only has a version range of 0-6.1, which can confuse users. To make the scores more comparable, the calculator will display a scaled score alongside the original score, which looks as follows: 3.4 → 5.7

The radar chart on the right visualizes these scores. Vectors can be excluded from this chart using the eye icon in the vector list. A thicker and dotted line means that the vector is currently selected. The selected vector will also be displayed below the radar chart. Toggle the switch at the top of the radar chart to split it into multiple charts for each CVSS version, improving readability when many vectors are present.
CVSS:4.0
CVSS:4.0 differs from previous versions in several ways:
  • Only a Base Score is provided, which is used on every axis of the radar chart.
  • During calculation, a MacroVector is calculated from the vector, which is a representation of the vector with a reduced amount of dimensions. This MacroVector is displayed below the radar chart if a CVSS:4.0 vector is selected.
  • The supplemental metrics are purely informational and do not affect the calculated score.
Further notes
More can be read about the CVSS standard at:
  • CVSS:2.0 documentation (https://www.first.org/cvss/v2/guide)
  • CVSS:3.1 documentation (https://www.first.org/cvss/v3.1/specification-document)
  • CVSS:4.0 documentation (https://www.first.org/cvss/v4.0/specification-document)

Changelog

Version 1.0.9 2024-01-24
  • Changed user guide icons to question marks .
  • Adjusted height and position of footer elements to appear vertically centered and to be more useful.
  • Fixed Attack Vector (AV) user guide answer to be listed under the incorrect response.
  • Vector names will now be displayed in cursive if: there is a CVE name in the name field AND the official NVD vectors could be fetched for that CVE AND the relevant fetched vector is different from the current vector.
    Right-clicking the name will reset the vector to the official fetched one.
  • The NVD fetching indicator will now properly handle multiple vectors being fetched from different sources.
Version 1.0.8 2024-01-23
  • Undefined metrics buttons are displayed more narrow to discourage setting base metrics to these values and to save space for the actual components.
  • Setting base metrics to ND or X will highlight them in red to make it clearer that these values are not valid as base metrics.
  • A vector can be reset to be undefined by shift-clicking the remove vector button.
  • The CVSS:4.0 nomenclature will now be displayed in the macro vector explanation card.
  • Added a user guide for the base components for vector versions 3.1 and 4.0. based on their respective specification documents.
Version 1.0.7 2024-01-22
  • Component names can now be clicked to copy them to clipboard.
  • Added a vector copy button to copy the vector string. By holding any modifier key while clicking it, only components will be copied that have been modified since the vectors have been added to the page (e.g. when opened via link or vectors have been pasted into the list).
Version 1.0.6 2024-01-19
  • Scrollbars on details table will now be hidden on larger devices.
  • "Usage Guide" and "Project on GitHub" links on initial page layout will now be displayed elsewhere (inside other text and in footer).
Version 1.0.5 2024-01-17
  • Added support for a b64gzip parameter that accepts a gzip compressed and base64 encoded string that it will decode and extract URL parameters from, replacing all existing URL parameters that are present in the encoded string.
    Using this is usually not worth it if only one vector is passed, but with three or more vectors, a significant reduction of more than 40-50% can be achieved.
  • Added a button to clear all vectors to the vector list.
  • Added a link to the publicly available GitHub repository, where the implementation of the CVSS standard used in this calculator and the source code of this page can be found.
  • Improved detection of CVSS names for vectors copied from a Vulnerability Assessment Dashboard.
Version 1.0.4 2024-01-16
  • Added a limit of 25 vectors to the vector list.
  • Improved mobile experience by adding more screen size checks and adding overflow scrolling behaviours.
Version 1.0.3 2024-01-15

    • Added a multi-CVSS input modal where multiple vectors can be pasted to bulk import them.
    • The CVSS and CVE input fields have been merged, they will now decide automatically what the input type is and act accordingly.
  • Added button to copy all active vectors into the clipboard.
  • Added button to copy current vector configuration as link into the clipboard.
Version 1.0.2 2024-01-12
  • Improved URL parameter interpretation code, preventing invalid vectors from causing unexpected behavior after loading the vectors from a URL.
  • Fixed CVSS:2.0 improperly using negative values for the impact score, especially regarding the radar chart.
  • Component button labels will now adapt to available space and potentially use abbreviations or only use the short name if not enough space is available.
  • Added a cve URL parameter that allows for loading CVSS from NVD when opening the page.
